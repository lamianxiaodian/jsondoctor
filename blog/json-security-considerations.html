JSON Security Considerations

While JSON is a simple data format, improper handling can lead to serious security vulnerabilities. This guide covers the most common security issues and how to prevent them.

1. JSON Injection Attacks

JSON injection occurs when untrusted data is inserted into JSON without proper escaping, potentially changing the structure or meaning of the data.

Vulnerable Code: 
// DANGEROUS: Manual JSON construction
const json = '{"name": "' + userInput + '"}';

// Attacker input: hello", "admin": true, "x": "
// Result: {"name": "hello", "admin": true, "x": ""}

Safe Alternative

// SAFE: Use JSON.stringify
const json = JSON.stringify({name: userInput});

2. JSON Hijacking

JSON hijacking is an attack where a malicious site reads JSON data from another site by exploiting how browsers handle script tags.

Prevention

// Always serve sensitive JSON with proper Content-Type
Content-Type: application/json

// Or prefix JSON with "while(1);" or similar
while(1);{"secret": "data"}

// Client removes prefix before parsing
const data = JSON.parse(response.substring(9));

3. Cross-Site Request Forgery (CSRF) with JSON

Attackers can sometimes trick browsers into sending JSON requests to vulnerable endpoints.

Prevention

  • Use CSRF tokens for state-changing operations
  • Check the Content-Type header
  • Require custom headers that browsers can't send cross-origin
// Server-side check
app.post('/api/data', (req, res) => {
  // Verify Content-Type
  if (req.headers['content-type'] !== 'application/json') {
    return res.status(400).send('Invalid Content-Type');
  }
  
  // Verify CSRF token
  if (req.headers['x-csrf-token'] !== session.csrfToken) {
    return res.status(403).send('Invalid CSRF token');
  }
  
  // Process request
});

4. Prototype Pollution

In JavaScript, merging objects with user-controlled keys can pollute the Object prototype.

Vulnerable Code: 
// DANGEROUS: Unsafe merge
function merge(target, source) {
  for (const key in source) {
    target[key] = source[key];
  }
}

// Attacker JSON: {"__proto__": {"admin": true}}

Safe Alternative

// SAFE: Block dangerous keys
function safeMerge(target, source) {
  const blocked = ['__proto__', 'constructor', 'prototype'];
  for (const key in source) {
    if (!blocked.includes(key)) {
      target[key] = source[key];
    }
  }
}

5. Denial of Service via JSON Parsing

Maliciously crafted JSON can cause excessive memory usage or CPU consumption during parsing.

Deep Nesting Attack

// Deeply nested JSON can cause stack overflow
{"a":{"a":{"a":{"a":...}}}}

Prevention

// Limit JSON size
app.use(express.json({ limit: '100kb' }));

// Limit nesting depth (custom parser)
function parseWithDepthLimit(json, maxDepth = 10) {
  let depth = 0;
  return JSON.parse(json, (key, value) => {
    if (typeof value === 'object' && value !== null) {
      depth++;
      if (depth > maxDepth) {
        throw new Error('Maximum depth exceeded');
      }
    }
    return value;
  });
}

6. Sensitive Data Exposure

Accidentally including sensitive data in JSON responses is a common security issue.

Prevention

// Never return full database objects
app.get('/api/user/:id', async (req, res) => {
  const user = await db.getUser(req.params.id);
  
  // SAFE: Explicitly select fields
  res.json({
    id: user.id,
    name: user.name,
    email: user.email
    // password, ssn, etc. are excluded
  });
});

7. Content-Type Validation

Always validate the Content-Type header when receiving JSON.

// Server-side validation
app.post('/api/data', (req, res) => {
  const contentType = req.headers['content-type'];
  
  if (!contentType || !contentType.includes('application/json')) {
    return res.status(415).json({
      error: 'Content-Type must be application/json'
    });
  }
  
  // Process JSON
});

8. Security Checklist

  • Always use JSON.stringify() instead of string concatenation
  • Validate and sanitize all JSON input
  • Use proper Content-Type headers
  • Implement CSRF protection for state-changing requests
  • Limit JSON payload size
  • Filter sensitive fields before serialization
  • Use HTTPS for JSON transmission
  • Keep JSON parsing libraries updated
