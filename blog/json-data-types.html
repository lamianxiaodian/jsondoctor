Understanding JSON Data Types

JSON supports six fundamental data types that form the building blocks of any JSON document. Understanding these types is essential for working effectively with JSON data.

1. String

Strings in JSON must be enclosed in double quotes. They can contain any Unicode characters and support escape sequences.

{
  "name": "Alice Johnson",
  "email": "alice@example.com",
  "message": "Hello, \"World\"!",
  "path": "C:\\Users\\Documents",
  "unicode": "Emoji: \u2764\ufe0f"
}

Escape Sequences

  • \" - Double quote
  • \\ - Backslash
  • \/ - Forward slash
  • \b - Backspace
  • \f - Form feed
  • \n - New line
  • \r - Carriage return
  • \t - Tab
  • \uXXXX - Unicode character

2. Number

JSON numbers can be integers or floating-point values. They do not distinguish between different number types like some programming languages do.

{
  "integer": 42,
  "negative": -17,
  "floating": 3.14159,
  "scientific": 2.998e8,
  "negativeExponent": 6.022e-23
}

Number Rules

  • No leading zeros (except for the value 0)
  • Can have a fractional part after a decimal point
  • Can have an exponent with 'e' or 'E'
  • Cannot use NaN or Infinity (not valid JSON)

3. Boolean

Boolean values are simple true/false literals, written in lowercase without quotes.

{
  "isActive": true,
  "hasPermission": false,
  "isLoggedIn": true
}

4. Null

The null value represents the absence of a value. It's useful for indicating missing or undefined data.

{
  "middleName": null,
  "phoneNumber": null,
  "avatar": null
}

5. Object

Objects are unordered collections of key-value pairs. Keys must be strings (in double quotes), and values can be any JSON data type.

{
  "user": {
    "id": 123,
    "profile": {
      "name": "Bob Smith",
      "age": 28,
      "address": {
        "street": "123 Main St",
        "city": "New York"
      }
    }
  }
}

6. Array

Arrays are ordered collections of values. Values can be of mixed types, including nested arrays and objects.

{
  "numbers": [1, 2, 3, 4, 5],
  "mixed": [1, "two", true, null, {"key": "value"}],
  "nested": [[1, 2], [3, 4], [5, 6]],
  "users": [
    {"name": "Alice", "age": 25},
    {"name": "Bob", "age": 30}
  ]
}

Type Checking in Different Languages

// JavaScript
typeof jsonValue === "string"
typeof jsonValue === "number"
typeof jsonValue === "boolean"
jsonValue === null
typeof jsonValue === "object" // for objects and arrays
Array.isArray(jsonValue) // specifically for arrays

// Python
type(value) == str
type(value) == int or type(value) == float
type(value) == bool
value is None
type(value) == dict
type(value) == list

Best Practices

  • Use appropriate types: don't store numbers as strings
  • Be consistent with null vs. omitted fields
  • Use arrays for ordered data, objects for keyed data
  • Keep nesting levels reasonable for readability
