JSON with TypeScript: Type Safety and Best Practices

TypeScript brings type safety to JavaScript, and when working with JSON, this becomes even more valuable. In this guide, we'll explore how to effectively handle JSON in TypeScript projects, ensuring type safety throughout the process.

Basic JSON Parsing in TypeScript

// Basic parsing (no type safety)
const jsonString = '{"name": "John", "age": 30}';
const user = JSON.parse(jsonString);
// user is of type 'any' - no type safety

// Better approach: Type assertion
interface User {
  name: string;
  age: number;
}

const typedUser = JSON.parse(jsonString) as User;
// typedUser is now of type User

Type Guards for JSON

Type assertions are quick but not safe. For better type safety, use type guards:

interface User {
  name: string;
  age: number;
  email?: string;
}

function isUser(obj: any): obj is User {
  return (
    typeof obj === 'object' &&
    obj !== null &&
    typeof obj.name === 'string' &&
    typeof obj.age === 'number' &&
    (obj.email === undefined || typeof obj.email === 'string')
  );
}

const jsonString = '{"name": "John", "age": 30}';
const parsed = JSON.parse(jsonString);

if (isUser(parsed)) {
  // parsed is now type User
  console.log(parsed.name); // Safe
} else {
  console.error('Invalid user data');
}

Using JSON Schema with TypeScript

For complex data structures, use JSON Schema with TypeScript:

// Install dependencies:
// npm install json-schema-to-typescript

// user.schema.json
/*
{
  "$id": "user.schema.json",
  "type": "object",
  "properties": {
    "name": { "type": "string" },
    "age": { "type": "integer" },
    "email": { "type": "string", "format": "email" }
  },
  "required": ["name", "age"]
}
*/

// Generate TypeScript types
// npx json-schema-to-typescript user.schema.json > user.types.ts

// user.types.ts
/*
export interface User {
  name: string;
  age: number;
  email?: string;
}
*/

Handling Optional and Nullable Fields

interface User {
  id: number;
  name: string;
  email: string | null;
  address?: {
    street: string;
    city: string;
  };
}

const jsonString = '{"id": 1, "name": "John", "email": null}';
const user = JSON.parse(jsonString) as User;

// Safe access to optional fields
const city = user.address?.city;

// Safe access to nullable fields
const email = user.email ?? 'No email provided';

Generic JSON Parsing Function

function safeJsonParse(jsonString: string): T | null {
  try {
    return JSON.parse(jsonString) as T;
  } catch (error) {
    console.error('JSON parsing error:', error);
    return null;
  }
}

interface Product {
  id: number;
  name: string;
  price: number;
}

const productJson = '{"id": 1, "name": "Laptop", "price": 999}';
const product = safeJsonParse(productJson);

if (product) {
  console.log(product.name); // Safe
}

Working with Arrays

interface User {
  id: number;
  name: string;
}

function parseUsers(jsonString: string): User[] {
  try {
    const data = JSON.parse(jsonString);
    if (!Array.isArray(data)) {
      throw new Error('Expected array');
    }
    return data.map(item => ({
      id: item.id,
      name: item.name
    }));
  } catch (error) {
    console.error('Error parsing users:', error);
    return [];
  }
}

const usersJson = '[{"id": 1, "name": "John"}, {"id": 2, "name": "Jane"}]';
const users = parseUsers(usersJson);

Advanced Type Safety with Zod

For even better type safety, consider using libraries like Zod:

// Install: npm install zod

import { z } from 'zod';

// Define schema
const UserSchema = z.object({
  name: z.string(),
  age: z.number(),
  email: z.string().email().optional(),
});

// Infer TypeScript type from schema
type User = z.infer;

// Parse and validate
const jsonString = '{"name": "John", "age": 30}';

try {
  const user = UserSchema.parse(JSON.parse(jsonString));
  // user is now type User with runtime validation
  console.log(user.name);
} catch (error) {
  console.error('Validation error:', error);
}

// Safe parsing (doesn't throw)
const result = UserSchema.safeParse(JSON.parse(jsonString));
if (result.success) {
  console.log(result.data.name);
} else {
  console.error(result.error);
}

TypeScript and JSON Serialization

interface User {
  name: string;
  age: number;
  createdAt: Date;
}

const user: User = {
  name: "John",
  age: 30,
  createdAt: new Date()
};

// Serialization
const jsonString = JSON.stringify(user, (key, value) => {
  if (value instanceof Date) {
    return value.toISOString();
  }
  return value;
});

// Deserialization with type safety
function deserializeUser(json: string): User {
  const parsed = JSON.parse(json);
  return {
    ...parsed,
    createdAt: new Date(parsed.createdAt)
  };
}

const deserializedUser = deserializeUser(jsonString);

Best Practices

  • Always define types: Create interfaces for all JSON structures
  • Use type guards: Validate data at runtime
  • Handle errors: Always wrap JSON.parse() in try-catch
  • Use Zod or similar libraries: For runtime validation
  • Be explicit about optional fields: Use ? for optional properties
  • Use JSON Schema: For shared data contracts
  • Document your types: Add JSDoc comments to interfaces

Common Pitfalls

  • Overusing any: Defeats the purpose of TypeScript
  • Ignoring runtime validation: TypeScript only checks at compile time
  • Forgetting Date objects: They serialize to strings
  • Complex nested types: Can become hard to maintain
