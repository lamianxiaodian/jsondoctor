JSON Parse Error: 5 Common Causes & Quick Fixes

JSON parse errors are frustrating, but they're usually caused by simple issues that can be fixed quickly. In this guide, we'll explore the most common causes of JSON parse errors and provide step-by-step solutions to resolve them.

1. Syntax Errors

Error Message: "Unexpected token X in JSON at position N"

Common Causes

  • Missing quotes around property names
  • Single quotes instead of double quotes
  • Missing commas between items
  • Trailing commas
  • Mismatched brackets or braces
Quick Fix: 
// Before (Wrong)
{
  name: "John",
  'age': 30
}

// After (Correct)
{
  "name": "John",
  "age": 30
}

2. Invalid Unicode Characters

Error Message: "Unexpected token X in JSON at position N"

Common Causes

  • Unescaped special characters
  • Invalid Unicode sequences
  • Control characters
Quick Fix: 
// Before (Wrong)
{
  "message": "He said "Hello""
}

// After (Correct)
{
  "message": "He said \"Hello\""
}

3. Circular References

Error Message: "Converting circular structure to JSON"

Common Causes

  • Objects that reference themselves
  • Complex nested structures with circular dependencies
Quick Fix: 
// Before (Wrong)
const obj = {};
obj.self = obj;
JSON.stringify(obj); // Error

// After (Correct)
function replacer(key, value) {
  if (key === 'self') {
    return '[Circular]';
  }
  return value;
}
JSON.stringify(obj, replacer);

4. Incomplete JSON

Error Message: "Unexpected end of JSON input"

Common Causes

  • Truncated JSON response from API
  • Network issues causing incomplete data
  • Accidentally cut off JSON string
Quick Fix: 
// Before (Wrong)
const jsonString = '{"name": "John"'; // Missing closing brace

// After (Correct)
const jsonString = '{"name": "John"}'; // Complete JSON

// Or use error handling
try {
  const data = JSON.parse(jsonString);
} catch (error) {
  console.error('Invalid JSON:', error);
  // Handle error gracefully
}

5. Incorrect Content-Type

Error Message: Varies (often "Unexpected token < in JSON at position 0")

Common Causes

  • Server returning HTML instead of JSON
  • Incorrect Content-Type header
  • API returning error page instead of JSON
Quick Fix: 
// Check response before parsing
fetch('api.example.com/data')
  .then(response => {
    if (!response.ok) {
      throw new Error('API error');
    }
    const contentType = response.headers.get('content-type');
    if (!contentType || !contentType.includes('application/json')) {
      throw new Error('Expected JSON response');
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error('Error:', error));

Debugging Tools

Use these tools to quickly identify and fix JSON errors:

  • JSON Doctor: Our free online tool for validating and formatting JSON
  • JSONLint: Online JSON validator
  • Visual Studio Code: Built-in JSON validation
  • jq: Command-line JSON processor

Preventative Measures

  • Always validate JSON before parsing
  • Use try-catch blocks around JSON.parse()
  • Check API responses for errors before parsing
  • Use consistent JSON formatting
  • Validate user input before serializing to JSON

Conclusion

JSON parse errors are usually caused by simple syntax issues, invalid characters, or incomplete data. By understanding the common causes and following the quick fixes provided in this guide, you can resolve these errors quickly and efficiently. Remember to always validate your JSON before parsing and handle errors gracefully in your code.
